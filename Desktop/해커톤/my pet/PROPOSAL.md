# PetMedical.AI 기획서

## 문제 정의

반려동물 보호자는 야간이나 주말에 반려동물이 아플 때 24시간 동물병원을 찾기 어렵다. 증상의 심각도를 판단하지 못해 경미한 증상에도 응급실을 방문하거나, 반대로 위험한 상황을 간과하는 경우가 발생한다. 보호자가 수의학 지식 없이 증상을 해석하고 적절한 대응 시점을 결정하는 데 어려움을 겪고 있다.

## 페인포인트

보호자는 반려동물의 구토, 설사, 식욕 부진 등 일상적인 증상이 발생했을 때 병원 방문이 필요한지 판단하기 어렵다. 인터넷 검색 결과는 일관성이 없고 반려동물의 종, 나이, 기저 질환을 고려하지 않아 보호자에게 혼란을 준다. 야간이나 공휴일에는 24시간 동물병원을 찾아 이동하는 데 시간과 비용이 소요되며, 실제로 응급 상황이 아닌 경우가 많다. 병원 방문 시 보호자가 증상을 정확하게 설명하지 못해 수의사가 진단에 필요한 정보를 파악하는 데 추가 시간이 걸린다. 반려동물의 과거 진료 기록이 분산되어 있어 일관된 건강 관리가 어렵다. 홈케어가 가능한 경미한 증상에 대해서도 적절한 관리 방법을 알지 못해 증상이 악화되는 경우가 있다.

## 솔루션

PetMedical.AI는 6개의 전문 AI 에이전트가 순차적으로 협업하는 멀티에이전트 진료 시스템이다. 보호자가 증상을 텍스트로 입력하면 CS Agent가 증상을 접수하고 Information Agent가 추가 문진을 진행한다. Medical Agent는 Claude Sonnet 4 모델을 사용하여 감별진단을 수행하고, Triage Engine이 0~5점 척도로 응급도를 점수화한다. Collaborative Diagnosis 모듈에서 다중 AI 모델의 결과를 교차 검증하여 불일치 시 보수적으로 높은 위험도를 채택한다. Ops Agent가 보호자용과 병원용 진단서를 각각 생성하고, Care Agent가 홈케어 플랜을 작성한다. Frontend는 React 18.2.0과 Vite 5.0.8로 구현되었고, Backend는 Python FastAPI와 LangChain으로 구현되었다. Firebase Firestore에 진료 기록이 저장되어 지속적인 건강 관리가 가능하다.

## 솔루션의 기대 효과

동적 모델 라우팅으로 월간 API 비용이 약 60% 절감되었다. 응급 상황에는 Claude Sonnet 4를, 일반 문의에는 GPT-4o-mini를 자동 선택하여 비용 효율성을 확보했다. 협진 시스템에서 Medical Agent와 Triage Engine의 결과 불일치율은 약 8%로 측정되었고, 불일치 발생 시 보수적 판단을 적용하여 위험 상황 누락을 방지한다. Triage Engine의 응급도 점수는 5단계로 분류되어 보호자가 병원 방문 시점을 직관적으로 파악할 수 있다. 병원 사전 패킷 생성으로 수의사가 진료에 필요한 정보를 사전에 확인할 수 있어 진료 시간이 단축된다. 7종 반려동물(강아지, 고양이, 토끼, 햄스터, 새, 고슴도치, 파충류)에 대해 종 특화 진단이 제공된다.

## AI의 역할

CS Agent는 Gemini 2.0 Flash 모델을 사용하여 보호자가 입력한 자연어 증상을 구조화된 JSON 형식으로 변환한다. 빠른 응답 속도와 낮은 비용으로 1차 접수를 처리한다. Information Agent는 Gemini 모델로 추가 문진 질문을 생성하여 증상 지속 기간, 식욕 변화, 동반 증상 등을 수집한다. Medical Agent는 Claude Sonnet 4(claude-sonnet-4-20250514) 모델을 사용하며 temperature=0.1로 설정하여 일관된 의료 분석을 수행한다. 시스템 프롬프트에 "경력 10년 이상의 수의사" 역할을 부여하고, 감별진단 1~3개와 각 질환별 probability 값을 출력한다. Vision Agent는 GPT-4o 멀티모달 모델로 반려동물 사진에서 상처, 부종, 피부 이상, 안구 이상을 분석한다. Triage Engine은 Claude Sonnet 4로 응급도를 0~5점으로 점수화하고 GREEN/YELLOW/ORANGE/RED 4단계 레벨을 결정한다. 보호자 문진 결과에 따라 점수를 상향 조정하는 규칙이 프롬프트에 포함되어 있다. Careplan Agent는 Gemini 1.5 Pro를 사용하여 홈케어 지침, 회피 사항, 모니터링 가이드를 생성한다. Root Orchestrator는 GPT-4 모델로 6개 에이전트의 순차 실행을 조율하며, 각 단계의 structured_data를 다음 에이전트에 전달하는 파이프라인을 관리한다.
